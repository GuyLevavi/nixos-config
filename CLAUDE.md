# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```nushell
rb       # sudo nixos-rebuild switch --flake /etc/nixos#nixbox  (apply all changes)
update   # sudo nix flake update /etc/nixos  (bump flake.lock, then run rb)
gcold    # sudo nix-collect-garbage --delete-older-than 14d
nsh ripgrep fd  # temporary shell with packages (nix-shell -p)
```

```bash
# Debug a failed rebuild:
sudo nixos-rebuild switch --flake /etc/nixos#nixbox --show-trace

# Roll back:
sudo nixos-rebuild switch --rollback

# List generations:
sudo nix-env --list-generations --profile /nix/var/nix/profiles/system
```

Inside `nix-shell`, bash does NOT auto-exec nushell (the `IN_NIX_SHELL` guard in `.bashrc` prevents this).

## Architecture

```
flake.nix                    # entry point — pins nixpkgs (unstable), home-manager, catppuccin/nix
home.nix                     # home-manager user config — shared across all machines
hosts/nixbox/
  configuration.nix          # system-level config (Intel desktop, Hyprland, greetd, Podman, fonts)
  hardware-configuration.nix # generated by nixos-generate-config; do not edit by hand
config/                      # dotfiles, read by home.nix at build time
  hypr/hyprland.conf         # Hyprland config (read via builtins.readFile)
  mako/config                # notification layout (colors injected by catppuccin-nix)
  wofi/{config,style.css}    # launcher config
  nvim/                      # LazyVim config (symlinked to ~/.config/nvim)
DECISIONS.md                 # detailed rationale for every architectural decision
```

**Key split:** `hosts/nixbox/configuration.nix` = system-level (boot, hardware, services, users). `home.nix` = user environment (programs, dotfiles, theming). Changes to either require `rb` to apply.

## Critical Patterns

### Catppuccin theming — use modules, not home.file

All theming goes through `catppuccin/nix` flake module. `catppuccin.flavor = "mocha"` and `catppuccin.accent = "mauve"` are set once in `home.nix`. Per-app modules (`catppuccin.kitty.enable`, `catppuccin.waybar.enable`, etc.) inject colors at build time.

**CRITICAL:** A catppuccin module only fires when `programs.<name>.enable = true` is also set. Installing a package via `home.packages` or `home.file.source` bypasses the module hook — theming will not be applied.

**Waybar and Hyprland** must use `programs.waybar` and `wayland.windowManager.hyprland` respectively (not `home.file`) because catppuccin-nix injects into those module hooks. `config/waybar/` dotfiles were deleted — Waybar is entirely configured inline in `home.nix`.

### LSP servers — Nix provides, Mason does not install

All LSP servers, formatters, and linters are in `programs.neovim.extraPackages` in `home.nix`. Mason's `ensure_installed` is overridden to `{}` in `config/nvim/lua/plugins/lsp.lua` to prevent it from downloading FHS-dependent binaries (they fail on NixOS). `:Mason` shows Nix-provided servers as "not installed" — this is cosmetic only; do not run `:MasonInstall` for them.

Active LSPs: `nixd` (Nix, flake-aware), `lua-language-server`, `typescript-language-server`, `pyright`. Formatters: `nixpkgs-fmt`, `stylua`, `black`.

### Shell setup

Bash is the PAM login shell (required for systemd/emergency mode). On interactive startup, `.bashrc` execs nushell. The guard `[[ -z "$IN_NIX_SHELL" ]]` prevents this inside `nix-shell`. Nushell aliases live in `configFile.text` (config.nu), not `envFile.text` (env.nu — aliases are invalid there).

### Adding packages

- User apps → `home.packages` in `home.nix`
- Programs needing catppuccin theming → `programs.<name>.enable = true` in `home.nix`
- System daemons / hardware drivers → `hosts/nixbox/configuration.nix`
- New LSP/formatter → `programs.neovim.extraPackages` in `home.nix`

### uwsm + greetd

`programs.uwsm.enable = true` silently sets `services.dbus.implementation = "broker"`. If enabling uwsm for the first time, use `nixos-rebuild boot` (not `switch`) — the running D-Bus daemon cannot be hot-swapped. `wayland.windowManager.hyprland.systemd.enable` must stay `false` because uwsm manages the systemd session.

## Notes

- `system.stateVersion` and `home.stateVersion` are `"25.05"` — **do not change**
- Zoxide uses `--cmd cd` to override the built-in `cd` in nushell (no alias needed)
- `DOCKER_HOST` points to rootless Podman socket; activate with `systemctl --user enable --now podman.socket`
- Chrome Catppuccin theme must be installed manually from the Web Store (extensions can't be declared for Chrome)
- For context on why any decision was made, see `DECISIONS.md`
