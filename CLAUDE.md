# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```nushell
rb       # sudo nixos-rebuild switch --flake /etc/nixos#(hostname)  — dynamic, works on any machine
update   # sudo nix flake update /etc/nixos  (bump flake.lock, then run rb)
gcold    # sudo nix-collect-garbage --delete-older-than 14d
nsh ripgrep fd  # temporary shell with packages (nix-shell -p)
```

```bash
# Debug a failed rebuild:
sudo nixos-rebuild switch --flake /etc/nixos#gamingbox --show-trace

# Roll back:
sudo nixos-rebuild switch --rollback

# List generations:
sudo nix-env --list-generations --profile /nix/var/nix/profiles/system
```

Inside `nix-shell`, bash does NOT auto-exec nushell (the `IN_NIX_SHELL` guard in `.bashrc` prevents this).

## Architecture

```
flake.nix                       # entry point — pins nixpkgs (unstable), home-manager, catppuccin/nix
home/
  base.nix                      # headless user config — shell, neovim, git, CLI tools
  gui.nix                       # GUI layer — imports base.nix, adds Hyprland + graphical apps
  gamingbox.nix                 # gamingbox overrides — imports gui.nix, NVIDIA env, 2K display
hosts/
  common/laptop.nix             # shared system config for all laptops (boot, nix-ld, nix settings…)
  nixbox/configuration.nix      # Intel desktop — imports common/laptop.nix
  gamingbox/configuration.nix   # NVIDIA RTX 4060 laptop — imports common/laptop.nix
  */hardware-configuration.nix  # generated by nixos-generate-config; do not edit by hand
config/
  hypr/hyprland.conf            # Hyprland config (read via builtins.readFile in gui.nix)
  vscode/settings.json          # VSCode settings seed (copied once on first activation)
  nvim/                         # LazyVim config (symlinked to ~/.config/nvim)
DECISIONS.md                    # detailed rationale for every architectural decision
AGENTS.md                       # agent-specific guidance (read this too)
```

**Machines:** `nixbox` (Intel desktop) · `gamingbox` (NVIDIA RTX 4060 laptop). Both use `hosts/common/laptop.nix` for shared system config. `rb` uses `hostname` dynamically — no need to specify the target by hand.

## Critical Patterns

### Catppuccin theming — use modules, not home.file

All theming goes through `catppuccin/nix`. `catppuccin.flavor = "mocha"` and `catppuccin.accent = "mauve"` set once in `gui.nix`. Per-app modules inject colors at build time.

**CRITICAL:** A catppuccin module only fires when `programs.<name>.enable = true` is also set. Installing via `home.packages` or `home.file.source` bypasses the hook — theming will not apply.

**Waybar and Hyprland** use `programs.waybar` / `wayland.windowManager.hyprland` (not `home.file`). Waybar is configured entirely inline in `gui.nix`.

### LSP servers — Nix provides, Mason does not install

LSP servers and formatters are in `programs.neovim.extraPackages` in `lazyvim.nix`. Mason's `ensure_installed` is overridden to `{}`. Do not run `:MasonInstall` — FHS binaries fail on NixOS. `:Mason` showing servers as "not installed" is cosmetic only.

`extraPackages` only adds to neovim's PATH wrapper — tools also needed by **VSCode** must be separately in `home.packages` in `gui.nix` (e.g. `ruff`, `nixd`, `nixpkgs-fmt`).

### Shell setup

Bash is the PAM login shell. On interactive startup, `.bashrc` execs nushell. Nushell aliases live in `configFile.text` (config.nu), not `envFile.text` (env.nu).

**`home.sessionVariables` does NOT work for nushell** — it generates a `.sh` file nushell never sources. Set env vars in `programs.nushell.envFile.text` instead.

### Adding packages

| What | Where |
|---|---|
| User CLI app | `home.packages` in `home/base.nix` |
| User GUI app | `home.packages` in `home/gui.nix` |
| App needing catppuccin theming | `programs.<name>.enable = true` in the appropriate file |
| System daemon / hardware driver | `hosts/<machine>/configuration.nix` or `hosts/common/laptop.nix` |
| New LSP/formatter (neovim) | `programs.neovim.extraPackages` in `lazyvim.nix` |
| Tool also needed by VSCode | additionally in `home.packages` in `gui.nix` |
| VSCode extension | `programs.vscode.extensions` in `gui.nix` |

### pip compiled extensions (gamingbox)

Two mechanisms required — see `AGENTS.md` and Decision 31 in `DECISIONS.md` for full details.
Short version: `programs.nix-ld` in `hosts/common/laptop.nix` fixes FHS entry-point binaries;
`LD_LIBRARY_PATH` in `programs.nushell.envFile.text` in `home/gamingbox.nix` fixes `dlopen()`.

### VSCode settings

`~/.config/Code/User/settings.json` is writable (owned by VSCode). Seeded once from
`config/vscode/settings.json` on first `rb`. Edit live in UI or the file directly.
Tool paths must be **absolute** (`/etc/profiles/per-user/gl/bin/ruff`), not tilde paths.

### uwsm + greetd

`programs.uwsm.enable = true` silently sets `services.dbus.implementation = "broker"`. If enabling
uwsm for the first time, use `nixos-rebuild boot` — the running D-Bus daemon cannot be hot-swapped.
`wayland.windowManager.hyprland.systemd.enable` must stay `false` (uwsm manages the session).

## Notes

- `system.stateVersion` and `home.stateVersion` are `"25.05"` — **do not change**
- Zoxide uses `--cmd cd` to override the built-in `cd` in nushell (no alias needed)
- `DOCKER_HOST` points to rootless Podman socket; activate with `systemctl --user enable --now podman.socket`
- Chrome Catppuccin theme must be installed manually from the Web Store
- For context on why any decision was made, see `DECISIONS.md`
- For agent-specific patterns and gotchas, see `AGENTS.md`
