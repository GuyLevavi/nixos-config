# nixos-config

Guy's NixOS configuration. Flakes + Home Manager.

## Structure

```
.
├── flake.nix                        # entry point, pins nixpkgs + home-manager
├── flake.lock                       # auto-generated, commit this
├── home.nix                         # user environment (shared across machines)
├── hosts/
│   ├── nixbox/
│   │   ├── configuration.nix        # Intel desktop system config
│   │   └── hardware-configuration.nix  # generated by nixos-generate-config (gitignore or commit)
│   └── winbox/                      # NVIDIA machine (future)
│       ├── configuration.nix
│       └── hardware-configuration.nix
└── config/                          # dotfiles symlinked via home.nix
    ├── hypr/
    │   └── hyprland.conf
    ├── waybar/
    ├── nvim/                        # LazyVim config (clone/symlink here)
    └── mako/
```

## Usage

```bash
# Apply changes
sudo nixos-rebuild switch --flake /etc/nixos#nixbox

# Aliased in nushell as:
rb

# Update all flake inputs (nixpkgs, home-manager)
sudo nix flake update /etc/nixos
rb

# Temporary shell with a package (nothing installed permanently)
nix-shell -p ripgrep

# Clean old generations (keep last 2 weeks)
sudo nix-collect-garbage --delete-older-than 14d

# Roll back to last working generation
sudo nixos-rebuild switch --rollback

# List generations
sudo nix-env --list-generations --profile /nix/var/nix/profiles/system

# Debug failed rebuild
sudo nixos-rebuild switch --flake /etc/nixos#nixbox --show-trace
```

## First-time setup

```bash
# After graphical installer, at TTY:
nix-shell -p git

# Clone this repo into /etc/nixos (or copy your files there)
sudo cp -r ~/nixos-config/* /etc/nixos/
cd /etc/nixos
git init && git add .

# First rebuild
sudo nixos-rebuild switch --flake /etc/nixos#nixbox
```

## Notes

- `system.stateVersion` and `home.stateVersion` — set at install time, **do not change**
- `hardware-configuration.nix` is machine-specific, generated by the installer
- Nushell is launched interactively from bash — bash stays as the PAM login shell
- LazyVim: manage plugins via lazy.nvim as usual; Nix only provides the runtime deps (LSPs, formatters, ripgrep etc.)
- Catppuccin Mocha is applied to: kitty, fzf. Apply it in LazyVim via the `catppuccin/nvim` plugin.
